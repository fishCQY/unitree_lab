# 奖励权重与 σ 调节指南

## 1. 指数核奖励的数学结构

在 unitree_lab 和 bfm_training 中，几乎所有跟踪类奖励都采用**指数核**（exponential kernel）形式：

```python
reward = exp(-error / σ²)
```

其中 `error` 是误差的平方和（或均方），`σ` 是传入的 `std` 参数。

最终总奖励为：

```
r_total = Σ (weight_i × exp(-error_i / σ_i²))
```

对应代码中的实现，例如线速度跟踪：

```python
# track_lin_vel_xy_yaw_frame_exp
lin_vel_error = torch.sum(torch.square(v_cmd[:, :2] - v_robot[:, :2]), dim=1)
return torch.exp(-lin_vel_error / std**2)
```

关节位置跟踪：

```python
# tracking_joint_pos_exp
joint_pos_error = torch.square(joint_pos_target - joint_pos).mean(dim=-1)
return torch.exp(-joint_pos_error / std**2)
```

关键点跟踪：

```python
# tracking_key_points_exp
body_pos_error = torch.square(key_points_target - keypoints).sum(dim=-2).mean(dim=-1).mean(dim=-1)
return torch.exp(-body_pos_error / std**2)
```

## 2. σ 的作用——控制"容忍度"

σ 决定了奖励函数对误差的**敏感程度**，即"机器人做到什么程度算好"。

### 2.1 σ 对奖励值的影响

以线速度跟踪为例（σ²=0.15）：

| 速度误差 (m/s) | error = ‖v_cmd - v‖² | reward = exp(-error / 0.15) |
|----------------|----------------------|-----------------------------|
| 0.0 | 0.0 | **1.00**（完美跟踪） |
| 0.2 | 0.04 | **0.77** |
| 0.4 | 0.16 | **0.34** |
| 0.6 | 0.36 | **0.09** |
| 1.0 | 1.00 | **0.001**（几乎为零） |

对比 σ²=0.25 时，同样 0.4 m/s 误差：

| σ² | reward |
|----|--------|
| 0.15 | 0.34（严格） |
| 0.25 | 0.53（宽松） |

### 2.2 σ 的直觉理解

```
σ 小 (0.15):  ___/‾\___    尖峰，只有精确跟踪才能拿高分
σ 大 (0.5):   __/‾‾‾‾\__   宽峰，大致跟上就能拿不错的分
```

**σ 越小，曲线越陡，对误差越不容忍；σ 越大，曲线越平缓，对误差越宽容。**

### 2.3 σ 的物理含义

当误差恰好等于 σ 时，`reward = exp(-1) ≈ 0.37`。可以把 σ 理解为"及格线"——误差小于 σ 的策略能拿到 0.37 以上的分，误差大于 σ 的策略接近零分。

---

## 3. 权重 (weight) 的作用——控制"优先级"

权重决定了该奖励项在总奖励中的**相对重要性**。

### 3.1 权重如何影响策略决策

考虑 AMP locomotion 的三个核心奖励：

```
r_total = 2.5 × exp(-vel_error / 0.15)    # 线速度跟踪
        + 1.5 × exp(-angvel_error / 0.5)   # 角速度跟踪
        - 0.1 × action_rate²               # 动作平滑
```

当策略需要在"快速变速"和"平稳动作"之间权衡时，2.5 vs 0.1 的权重比告诉策略："跟上速度命令比动作平滑重要 25 倍"。

### 3.2 正奖励与负惩罚的平衡

```
AMP Locomotion 权重结构：
  正奖励总和: 2.5 + 1.5 = 4.0
  负惩罚总和: -0.1 + -1.0 + -2.0 = -3.1
  比值: 4.0 / 3.1 ≈ 1.3（正奖励略占优）
```

正奖励总和应大于负惩罚总和，否则策略倾向"什么都不做"来避免惩罚。

---

## 4. σ 与 weight 的本质区别

| 操作 | 效果 | 类比 |
|------|------|------|
| **增大 weight** | 奖励的绝对数值按比例放大，曲线形状不变 | 考试中这道题分值更高 |
| **减小 σ** | 曲线变陡，只有做得很好才有高分，但最大值不变 (=1) | 这道题的评分标准更严格 |

一个具体对比：

```
方案A: weight=1.0, σ²=0.01  → 精确跟踪时 reward=1.0，误差稍大就骤降到~0
方案B: weight=10.0, σ²=1.0  → 精确跟踪时 reward=10.0，哪怕误差较大仍有可观reward
```

- **方案 A**：鼓励"精确"但总分贡献小（高标准低分值）
- **方案 B**：鼓励"尽力做"且做不好也给部分分（低标准高分值）

## 5. weight/σ²——真正的关键指标

奖励对误差的梯度为：

```
∂reward/∂error = -weight / σ² × exp(-error / σ²)
```

在误差 = 0 处（完美跟踪），梯度大小为 `weight/σ²`。这个比值决定了**策略在改善该项 1 单位误差时获得多少奖励增益**。

### 5.1 当前配置的 weight/σ² 分析

**AMP Locomotion（bfm_training 风格）：**

| 奖励项 | weight | σ² | weight/σ² | 梯度强度 |
|--------|--------|------|-----------|----------|
| 线速度跟踪 | 2.5 | 0.15 | **16.7** | 强 |
| 角速度跟踪 | 1.5 | 0.5 | **3.0** | 中 |

**Mimic（bfm_training 风格）：**

| 奖励项 | weight | σ² | weight/σ² | 梯度强度 |
|--------|--------|------|-----------|----------|
| 关键点跟踪 (world) | 4.0 | 0.125 | **32.0** | 最强 |
| 关键点跟踪 (body) | 3.0 | 0.125 | **24.0** | 极强 |
| 根速度跟踪 | 4.0 | 0.25 | **16.0** | 强 |
| 根角速度跟踪 | 1.0 | 1/12 | **12.0** | 强 |
| 关节位置跟踪 | 1.0 | 0.25 | **4.0** | 较弱 |

### 5.2 梯度强度的含义

`weight/σ²` 越大，策略在该维度上收到的"纠正推力"越强。上表说明：

- Mimic 任务中，**关键点空间位置**是策略最被强烈推动去改善的目标（32.0）
- **关节角度**的梯度最弱（4.0），因为不同的关节角度组合可以产生相同的末端位置（运动学冗余），策略有自由度

## 6. 调节 σ 的准则

### 6.1 根据物理量的典型误差范围设定

| 物理量 | 典型误差范围 | 推荐 σ² | σ 处 reward |
|--------|-------------|---------|-------------|
| 线速度 (m/s) | 0.1 - 0.5 | 0.15 | 0.39 m/s 时 ≈ 0.37 |
| 角速度 (rad/s) | 0.2 - 1.0 | 0.5 | 0.71 rad/s 时 ≈ 0.37 |
| 关节角度 (rad) | 0.05 - 0.3 | 0.25 | 0.5 rad 时 ≈ 0.37 |
| 关键点位置 (m) | 0.02 - 0.1 | 0.125 | 0.35 m 时 ≈ 0.37 |
| 根高度 (m) | 0.02 - 0.1 | 1/12 ≈ 0.083 | 0.29 m 时 ≈ 0.37 |
| 根姿态 (quat error) | 0.1 - 0.5 | 0.16 | 0.40 时 ≈ 0.37 |

**经验法则：令 σ² ≈ "可接受的最大误差的平方"。**

### 6.2 训练阶段的 σ 策略

| 训练阶段 | σ 策略 | 原因 |
|----------|--------|------|
| 早期（策略随机） | 较大 σ | 宽容的奖励让策略有梯度可循，不会因为太差得不到任何正反馈 |
| 中期（基本收敛） | 适中 σ | 保持有效梯度的同时提高精度要求 |
| 后期（精调） | 较小 σ | 收紧标准，迫使策略在细节上提升 |

---

## 7. 调节 weight 的准则

### 7.1 按优先级分配

```
总原则：最重要的目标 > 次要约束 > 正则化项
```

**AMP Locomotion 的权重层次：**
```
速度跟踪 (2.5 + 1.5 = 4.0)  >>  安全约束 (-2.0 + -1.0 = -3.0)  >>  平滑 (-0.1)
```

**Mimic 的权重层次：**
```
关键点跟踪 (4.0 + 3.0 = 7.0)  >  根速度 (4.0 + 1.0 = 5.0)  >  关节 (1.0 + 1.0 = 2.0)  >>  惩罚 (-0.3 + -1.0 = -1.3)
```

### 7.2 权重设定经验法则

1. **正奖励总和 > 负惩罚总和**（否则策略"不敢动"）
2. **单项惩罚 < 主奖励的 50%**（否则策略过度保守）
3. **各正奖励的权重比例 ≈ 它们的重要性比例**
4. **惩罚类权重的绝对值不宜过大**，否则策略会钻空子（比如倒地不动来避免所有惩罚）

## 8. 常见问题诊断

| 现象 | 原因 | 解决方案 |
|------|------|----------|
| 某项 reward 始终 ≈ 0 | σ 太小，误差远超有效范围 | 增大 σ² 到 2-4 倍 |
| 某项 reward 始终 ≈ 1 | σ 太大，已饱和无梯度 | 减小 σ² 到 1/2 - 1/4 |
| 策略跟踪好但动作抖动 | action_rate weight 太小 | 增大到 -0.1 ~ -0.3 |
| 策略完全不敢动 | 惩罚总和远大于正奖励 | 降低惩罚权重或增大正奖励 |
| 策略只做某一个目标 | 该项 weight/σ² 远大于其他 | 平衡各项的 weight/σ² 比值 |
| 训练初期 reward 不增长 | 所有 σ 都太小 | 先用大 σ 让策略"入门"，后期再收紧 |
| 策略在某个局部最优卡住 | 惩罚太强或 σ 过严导致探索不足 | 暂时放松 σ 或降低惩罚 |
| 训练后期精度不再提升 | σ 已经太大，奖励饱和 | 减小 σ 迫使精细化 |

## 9. 两个代码库的调参对比

### 9.1 AMP Locomotion

| 参数 | unitree_lab (原始) | bfm_training | 调整方向 |
|------|-------------------|-------------|----------|
| 线速度 σ² | 0.25 | **0.15** | 收紧精度 |
| 角速度 σ² | 0.25 | **0.5** | 放松精度 |
| 线速度 weight | 1.0 | **2.5** | 提高优先级 |
| 角速度 weight | 1.0 | **1.5** | 提高优先级 |
| action_rate weight | -0.01 | **-0.1** | 增强平滑 |

bfm_training 对线速度更严格（σ 更小）但对角速度更宽松（σ 更大），说明其策略优先保证直线行走的精度，对转弯的精度要求较低。

### 9.2 Mimic (DeepMimic)

| 参数 | unitree_lab (原始) | bfm_training | 调整方向 |
|------|-------------------|-------------|----------|
| 关键点 σ² | 0.01 | **0.125** | 大幅放松 |
| 关键点 weight | 1.0 | **4.0** | 大幅提高 |
| 关节位置 σ² | 0.1 | **0.25** | 放松 |
| action_rate weight | -0.01 | **-0.3** | 30 倍增强 |

bfm_training 把关键点的 σ² 放大了 12 倍但权重提高了 4 倍。这意味着：
- **旧配置**：σ²=0.01, weight=1.0 → weight/σ²=100，梯度极强但奖励几乎总是 0
- **新配置**：σ²=0.125, weight=4.0 → weight/σ²=32，梯度适中且策略能拿到有意义的分数

旧配置的问题是 σ 太小（0.1m），训练初期策略的关键点误差远超此范围，导致 reward≈0 没有梯度，相当于"评分标准太严格以至于所有人都是零分"。bfm_training 放宽了 σ 让策略能"入门"，同时加大 weight 确保精度提升时有足够的奖励激励。

## 10. 总结

| 要素 | 控制什么 | 关键数值 |
|------|----------|----------|
| **σ** | 容忍度（做到什么程度算好） | 误差 = σ 时 reward ≈ 0.37 |
| **weight** | 优先级（这个目标有多重要） | 正奖励总和应 > 负惩罚总和 |
| **weight/σ²** | 梯度强度（改善 1 单位误差获得多少增益） | 最重要的目标应有最大的 weight/σ² |

调参的核心思路：**先用大 σ 让策略学会基本行为，再逐步收紧 σ 追求精度；用 weight 确定各项的优先级排序；用 weight/σ² 检查梯度是否均衡。**
